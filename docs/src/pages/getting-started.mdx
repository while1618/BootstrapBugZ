## üê≥ Running the Project

To run the project locally, all you need is Docker. Follow these steps:

```bash
git clone https://github.com/while1618/bugzkit.git
cd bugzkit
docker-compose up --build -d
```

Access the application at these URLs:

- **UI**: [http://localhost:5173](http://localhost:5173)
- **API**: [http://localhost:8080/v1/users](http://localhost:8080/v1/users)
- **API Docs**: [http://localhost:8080/swagger-ui/index.html](http://localhost:8080/swagger-ui/index.html)

üí° **Note**: Email functionalities (e.g., account confirmation, password reset) won't work without configuring SMTP. Check the Environment Variables section and update the defaults with your setup.

## üõ†Ô∏è Development Setup

### üîß Prerequisites

To start development, ensure you have the following:

- [Gum](https://github.com/charmbracelet/gum)
- [Docker](https://www.docker.com/)
- [Java 21](https://openjdk.org/projects/jdk/21/)
- [Maven](https://maven.apache.org/)
- [Node.js 22](https://nodejs.org/)
- [pnpm](https://pnpm.io/)
- [PostgreSQL](https://www.postgresql.org/) (optional; can be run via Docker)
- [Redis](https://redis.io/) (optional; can be run via Docker)

### üì¶ Initialize Your Project

Create a new repository using this [template](https://github.com/new?template_name=bugzkit&template_owner=while1618) and run the initialization script:

```bash
./init.sh
```

The script will:

1. Ask for your project name.
2. Replace all instances of `bugzkit` with your project name.
3. Remove unnecessary files and folders, including `docs` and the script itself.

Once completed, commit the changes and start coding! üéâ

## üñ•Ô∏è Running the Backend

### üßë‚Äçüíª Running Backend Locally

If you prefer not to install PostgreSQL and Redis locally, you can use the `docker-compose-db-dev.yml` file in the `backend` directory:

```bash
docker-compose -f docker-compose-db-dev.yml up -d
```

After the services are running, start the application:

```bash
mvn clean install
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

Alternatively, use IntelliJ IDEA. A pre-configured `Application` run configuration is available‚Äîjust click and run! üõ†Ô∏è

### üê≥ Running Backend via Docker

For running the backend API using Docker, execute:

```bash
docker-compose -f docker-compose-api-dev.yml up --build -d
```

üí° **Note**: Email functionalities require SMTP configuration. Refer to the Environment Variables section for setup.

## üåê Running the Frontend

Ensure the backend API is up and running before starting the UI. Then, create a `.env` file in the root of the frontend directory with the following content:

```bash
# Public
PUBLIC_APP_NAME=your_app_name
PUBLIC_API_URL=http://localhost:8080/v1
# Private
JWT_SECRET=secret
```

To start the UI, run:

```bash
pnpm install
pnpm run dev
```

You're all set! üöÄ
